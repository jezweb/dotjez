# artifacts

The catch-all for session outputs that aren't a plan, audit, or piece of research: dated working notes, comparisons, design docs, dumps, brains-trust results. If it doesn't have an obvious home in `plans/`, `research/`, or `audits/`, it goes here.

One file per output, date in the filename when it matters. As it grows, group related outputs into sub-folders by topic (`artifacts/audits/`, `artifacts/dumps/`), once about three of a kind pile up. If something here turns out to matter across projects, move it up to your workspace `.jez/`.
